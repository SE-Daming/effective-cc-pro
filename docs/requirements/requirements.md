# 需求分析文档

> 仿猫眼电影票务平台 - 需求规格说明书

---

## 1. 项目概述

### 1.1 项目背景

开发一个仿猫眼/淘票票的电影票务平台，包含：
- **C 端小程序**：用户购票、观影体验
- **B 端管理后台**：影院运营、数据管理
- **AI 助手模块**：智能推荐、客服（待定）

### 1.2 目标用户

| 用户类型 | 说明 |
|----------|------|
| C 端用户 | 普通消费者，购票观影 |
| B 端用户 | 影院管理员、运营人员 |

### 1.3 毕设侧重点

**功能完整性** — 侧重业务逻辑完整性，功能全面

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2s |
| 接口响应时间 | < 500ms |
| 并发用户数 | 支持 100+ 并发 |

### 4.2 安全需求

| 需求 | 说明 |
|------|------|
| 数据传输加密 | HTTPS |
| 支付安全 | 微信支付官方接口 |
| 敏感信息保护 | 密码加密存储 |

### 4.3 兼容性

| 平台 | 要求 |
|------|------|
| 小程序 | 微信小程序 |
| Web 后台 | Chrome、Edge 主流浏览器 |

## 5. 功能优先级汇总

### 5.1 MVP 核心（必须实现）

**C 端：**
- 微信登录
- 电影列表、详情
- 影院列表、详情、排片
- 选座购票、订单管理、支付
- 优惠券：领券、购票抵扣 ⭐

**B 端：**
- 管理员登录
- 电影管理
- 影院管理、影厅管理
- 排片管理
- 订单管理、退款审核
- 优惠券管理 ⭐

### 5.2 扩展功能（时间充裕时）

**C 端：**
- 会员体系：积分、会员价、等级 ⭐
- 消息通知：取票、开映提醒 ⭐
- 影评功能
- 收藏功能
- 个性化推荐（AI 模块）

**B 端：**
- 会员管理 ⭐
- 消息管理 ⭐
- 角色权限
- 数据可视化
