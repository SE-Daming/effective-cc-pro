# AI 助手模块需求说明

> 独立 Spring Boot 服务，提供智能客服、智能推荐、Function Call 能力

---

## 1. 模块概述

### 1.1 核心定位

综合智能助手，具备三大核心能力：

| 能力 | 说明 |
|------|------|
| **智能客服** | 业务咨询、规则解释 |
| **智能推荐** | 电影、影院、优惠券推荐 |
| **Function Call** | 意图识别后调用业务接口完成操作 |

### 1.2 技术方案

| 技术选型 | 说明 |
|----------|------|
| Spring AI | AI 应用开发框架 |
| 百炼平台 API | 阿里云大模型平台 |
| HTTP | 调用主服务接口获取数据 |

### 1.3 服务架构

```
┌─────────────────────────────────────────────────────┐
│                    前端层                            │
│         ┌─────────────┐    ┌─────────────┐         │
│         │  Vue Web    │    │  UniApp     │         │
│         └──────┬──────┘    └──────┬──────┘         │
└────────────────┼──────────────────┼────────────────┘
                 │ HTTP             │ HTTP
                 ▼                  ▼
┌─────────────────────────────────────────────────────┐
│                   后端层                             │
│         ┌─────────────────────────────────┐         │
│         │      Spring Boot 主服务          │         │
│         │      (业务逻辑)                  │         │
│         └──────────────▲──────────────────┘         │
│                        │ HTTP                       │
│                        │                            │
│         ┌──────────────┴──────────────────┐         │
│         │    Spring Boot AI 助手模块       │         │
│         │      (智能对话 + 推荐调度)        │         │
│         └──────────────┬──────────────────┘         │
│                        │ API                         │
│                        ▼                            │
│         ┌─────────────────────────────────┐         │
│         │       百炼平台 API               │         │
│         │      (大模型能力)                │         │
│         └─────────────────────────────────┘         │
└─────────────────────────────────────────────────────┘
```

## 2. 智能客服能力

### 2.1 业务咨询

| 场景 | 示例问题 | AI 回答 |
|------|----------|---------|
| 电影查询 | "最近有什么科幻片上映？" | 列出当前热映的科幻电影 |
| 影院查询 | "附近有什么电影院？" | 列出附近影院及距离 |
| 排片查询 | "万达影院今天有什么电影？" | 列出该影院今日排片 |
| 票价查询 | "《流浪地球3》票价多少？" | 返回票价信息 |

### 2.2 规则解释

| 场景 | 示例问题 | AI 回答 |
|------|----------|---------|
| 退票规则 | "电影票怎么退？" | 解释退票时间限制、手续费规则 |
| 积分规则 | "积分怎么用？" | 解释积分获取、使用规则 |
| 会员规则 | "黄金会员有什么权益？" | 列出黄金会员权益 |
| 优惠券规则 | "这个券怎么用？" | 解释优惠券使用条件、有效期 |

## 3. 智能推荐能力

### 3.1 电影推荐

| 推荐场景 | 推荐逻辑 | 数据来源 |
|----------|----------|----------|
| 基于偏好 | 根据用户观影历史、评分推荐 | 用户历史订单、评分记录 |
| 基于热度 | 推荐当前热门电影 | 票房排行、评分排行 |
| 基于类型 | 用户说"推荐一部喜剧片" | 电影类型筛选 |
| 基于场景 | 用户说"适合情侣看的电影" | 电影标签匹配 |

### 3.2 影院推荐

| 推荐场景 | 推荐逻辑 | 数据来源 |
|----------|----------|----------|
| 基于位置 | 推荐距离最近的影院 | 用户位置、影院地址 |
| 基于评分 | 推荐评分最高的影院 | 影院评分数据 |
| 基于设施 | 推荐有 IMAX/Dolby 的影院 | 影院设施标签 |

### 3.3 优惠券推荐

| 推荐场景 | 推荐逻辑 | 数据来源 |
|----------|----------|----------|
| 即将过期 | 推荐用户即将过期的优惠券 | 用户优惠券列表 |
| 满足条件 | 用户购票时推荐可用优惠券 | 订单金额、优惠券规则 |
| 个性化 | 根据用户偏好推荐适合的券 | 用户画像、优惠券标签 |

## 4. Function Call 能力

### 4.1 智能购票

**用户意图示例：**
- "帮我买一张最近的科幻片"
- "帮我买明天下午《流浪地球3》的票"
- "帮我订两张万达影院晚上的票"

**执行流程：**

```
用户输入 → 意图识别 → 参数提取 → 调用接口 → 结果反馈
    │
    ▼
"帮我买一张最近的科幻片"
    │
    ▼
意图：购票
参数：{ genre: "科幻", quantity: 1, time: "最近" }
    │
    ▼
1. 调用电影列表接口 → 筛选科幻片
2. 调用排片接口 → 查询场次
3. 返回候选列表让用户确认
4. 用户选择场次、座位
5. 调用下单接口 → 完成购票
```

### 4.2 智能退票

**用户意图示例：**
- "帮我退掉明天的票"
- "我想退《流浪地球3》的票"

**执行流程：**

```
用户输入 → 意图识别 → 参数提取 → 调用接口 → 结果反馈
    │
    ▼
"帮我退掉明天的票"
    │
    ▼
意图：退票
参数：{ date: "明天" }
    │
    ▼
1. 调用订单查询接口 → 找到明天订单
2. 校验退票规则（开场前2小时）
3. 返回可退订单让用户确认
4. 调用退票接口 → 完成退票
```

### 4.3 查询订单

**用户意图示例：**
- "查一下我的订单"
- "我在万达影院的消费记录"

**执行流程：**

```
用户输入 → 意图识别 → 参数提取 → 调用接口 → 结果反馈
    │
    ▼
"查一下我在万达影院的消费记录"
    │
    ▼
意图：查询订单
参数：{ cinema: "万达影院" }
    │
    ▼
1. 调用订单查询接口 → 筛选影院
2. 返回订单列表
```

### 4.4 影院推荐（带查询）

**用户意图示例：**
- "帮我推荐附近评分最高的影院"
- "找个有 IMAX 的电影院"

**执行流程：**

```
用户输入 → 意图识别 → 参数提取 → 调用接口 → 结果反馈
    │
    ▼
"帮我推荐附近评分最高的影院"
    │
    ▼
意图：影院推荐
参数：{ location: "附近", sortBy: "评分" }
    │
    ▼
1. 调用影院列表接口 → 按距离筛选
2. 按评分排序
```

## 6. 对话交互设计

### 6.1 多轮对话示例

**场景：智能购票**

```
用户：帮我买一张科幻片
AI：好的，为您推荐以下科幻片：
    1. 流浪地球3 - 评分 9.2
    2. 三体 - 评分 8.8
    请问您想看哪一部？

用户：流浪地球3
AI：好的，《流浪地球3》，请问您在哪个影院观影？
    为您推荐附近影院：
    1. 万达影城（XX店）- 距离 1.2km
    2. CGV 影城（XX店）- 距离 2.5km

用户：万达影城
AI：好的，万达影城今日《流浪地球3》排片：
    1. 14:30 - IMAX厅
    2. 17:00 - 普通厅
    3. 19:30 - 杜比厅
    请选择场次。

用户：19:30
AI：好的，19:30 杜比厅，请问您需要选择哪个座位？
    [座位图展示]

用户：F排8座
AI：已为您选择 F排8座，票价 45 元。
    确认下单吗？

用户：确认
AI：订单创建成功！订单号：202505020001
    正在为您跳转支付页面...

[用户完成支付后]

AI：支付成功！取票码：886655，请凭此码到影院取票。
    电影开场前 30 分钟会提醒您，祝您观影愉快！
```

### 6.2 智能退票流程（含手续费告知）

```
用户：帮我退掉明天的票
AI：找到您的订单：《流浪地球3》明晚 19:30，票价 45 元
    开场前退票需收取 10% 手续费（4.5 元），实际退款 40.5 元。
    确认退票吗？

用户：确认
AI：退票成功！退款 40.5 元将在 1-3 个工作日原路返回。
```

### 6.3 错误处理

| 场景 | AI 回复 |
|------|---------|
| 无匹配电影 | "抱歉，没有找到相关电影，换个说法试试？" |
| 无场次 | "该影院暂无此电影排片，建议选择其他影院" |
| 座位已售 | "该座位已被选购，请重新选择" |
| 退票失败 | "该订单已开场，无法退票" |
| 锁座超时 | "选座超时，座位已释放，请重新选择" |
| 支付超时 | "支付超时，订单已取消，请重新下单" |

## 7. 技术实现方案

### 7.1 意图识别机制

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| **大模型识别（主）** | 通过 Prompt 引导大模型识别用户意图 | 复杂意图、模糊表达 |
| 关键词匹配（辅） | 预定义关键词规则快速匹配 | 明确指令（如"退票"、"查订单"） |

**Prompt 设计示例：**

```
你是一个电影票务助手。根据用户输入，识别用户意图并提取参数。

支持的意图：
- BOOK_TICKET: 购票（参数：电影名、影院、时间、数量）
- REFUND_TICKET: 退票（参数：订单号、时间）
- QUERY_ORDER: 查询订单（参数：状态、影院）
- SEARCH_MOVIE: 搜索电影（参数：类型、关键词）
- SEARCH_CINEMA: 搜索影院（参数：位置、设施）
- ASK_RULE: 咨询规则（参数：规则类型）

用户输入：{user_input}

请返回 JSON 格式：
{
  "intent": "意图名称",
  "parameters": { ... },
  "confidence": 0.95
}
```

### 7.2 上下文管理

| 方案 | 说明 |
|------|------|
| 存储方式 | Redis 存储 Session 上下文 |
| Session Key | `ai:session:{userId}:{sessionId}` |
| 过期时间 | 30 分钟无交互自动过期 |
| 存储内容 | 对话历史、当前意图、已提取参数 |

**Session 数据结构：**

```json
{
  "userId": 10001,
  "sessionId": "sess_xxx",
  "createdAt": "2025-05-02T10:00:00",
  "lastActiveAt": "2025-05-02T10:15:00",
  "currentIntent": "BOOK_TICKET",
  "extractedParams": {
    "movieName": "流浪地球3",
    "cinemaId": 5
  },
  "conversationHistory": [
    {"role": "user", "content": "帮我买一张科幻片"},
    {"role": "assistant", "content": "好的，为您推荐..."}
  ]
}
```

### 7.3 用户身份识别

| 场景 | 方式 |
|------|------|
| 小程序端 | 前端调用时在 Header 传递 `X-User-Id`（微信授权后获取） |
| Web 端 | 前端调用时在 Header 传递 `X-User-Id` + `X-Token` |
| AI 调用主服务 | AI 服务携带 `X-User-Id` 调用，主服务校验权限 |

**请求示例：**

```
POST /api/ai/chat
Headers:
  X-User-Id: 10001
  X-Token: xxx
Body:
  {
    "message": "帮我买一张科幻片",
    "sessionId": "sess_xxx"
  }
```

## 8. 完整接口清单

### 8.1 AI 服务调用主服务接口

| 接口 | 方法 | 用途 | 对应 Function |
|------|------|------|---------------|
| `/api/movies` | GET | 查询电影列表（支持分页） | search_movies |
| `/api/cinemas` | GET | 查询影院列表（支持分页） | search_cinemas |
| `/api/schedules` | GET | 查询排片信息 | get_schedules |
| `/api/schedules/{id}/seats` | GET | 获取座位图 | get_seats |
| `/api/seats/lock` | POST | 锁定座位 | lock_seats |
| `/api/seats/release` | POST | 释放座位 | release_seats |
| `/api/orders` | POST | 创建订单 | create_order |
| `/api/orders/{id}/pay` | POST | 发起支付 | pay_order |
| `/api/orders` | GET | 查询订单列表 | query_orders |
| `/api/orders/{id}/refund` | POST | 申请退票 | refund_order |
| `/api/coupons` | GET | 获取用户优惠券 | get_user_coupons |
| `/api/users/{id}/history` | GET | 获取用户观影历史 | get_user_history |
| `/api/users/{id}/profile` | GET | 获取用户画像 | get_user_profile |

## 11. 安全机制

### 11.1 越权校验

| 场景 | 校验方式 |
|------|----------|
| 查询订单 | AI 调用时携带 `X-User-Id`，主服务校验订单归属 |
| 退票操作 | 校验订单 `userId` 与当前用户一致 |
| 优惠券使用 | 校验优惠券归属用户 |

**主服务校验逻辑：**

```java
// 订单操作前校验
Order order = orderService.getById(orderId);
if (!order.getUserId().equals(currentUserId)) {
    throw new BusinessException("无权操作此订单");
}
```

### 11.2 敏感操作确认

| 操作 | 确认方式 |
|------|----------|
| 购票下单 | AI 文字确认 + 跳转支付页二次确认 |
| 退票退款 | AI 明确告知手续费，用户文字确认 |
| 使用优惠券 | AI 展示优惠金额，用户确认 |

### 11.3 Prompt 注入防护

| 防护措施 | 说明 |
|----------|------|
| 输入过滤 | 过滤特殊字符、SQL 关键词 |
| Prompt 隔离 | 用户输入不直接拼接到 Prompt，使用模板变量 |
| 输出校验 | 校验 AI 返回的 Function Call 参数合法性 |
| 权限限制 | Function Call 仅能操作当前用户数据 |

**Prompt 模板示例：**

```
你是一个电影票务助手。

【重要规则】
1. 你只能查询和操作当前用户（userId: {userId}）的数据
2. 对于退款等敏感操作，必须先告知用户规则并确认
3. 不要执行任何与票务无关的操作

用户输入：
{user_input}
```

## 12. 性能优化

### 12.1 Token 控制

| 策略 | 说明 |
|------|------|
| 历史截断 | 仅保留最近 10 轮对话历史 |
| 摘要压缩 | 长对话压缩为摘要后存储 |
| 意图缓存 | 相同意图复用历史参数提取结果 |

### 12.2 响应优化

| 策略 | 说明 |
|------|------|
| 流式输出 | 使用 SSE 流式返回 AI 回复，提升体验 |
| 并行调用 | 多个独立 Function Call 并行执行 |
| 接口缓存 | 电影列表、影院列表等数据缓存 |

### 12.3 限流策略

| 限流维度 | 规则 |
|----------|------|
| 用户级 | 每用户每分钟最多 20 次 AI 对话 |
| IP 级 | 同一 IP 每分钟最多 50 次请求 |

## 13. 异常处理

### 13.1 对话异常

| 场景 | 处理方式 |
|------|----------|
| Session 超时 | 提示"对话已超时，请重新开始" |
| 中断恢复 | 用户返回时检查 Session，未过期则恢复上下文 |
| 并发请求 | 同一 Session 仅处理一个请求，返回"请等待上一次请求完成" |
